Add Static Routes on Keenetic

Step-by-step guide: route traffic of specific websites through a VPN tunnel on a Keenetic router. Generate routes from domains with NetRoute Pro and import them on the router.

Prerequisites

  • Keenetic router with KeeneticOS 3.x or newer
  • Configured VPN tunnel (WireGuard, OpenVPN, IKEv2, or other)
  • NetRoute Pro Chrome extension installed
  • Access to the router's web UI (usually http://my.keenetic.net)

Step 1. Generate routes with NetRoute Pro

  1. Open the target website in Chrome
  2. Click the NetRoute Pro icon in your extensions
  3. Select the Linux platform (Keenetic accepts Linux-style commands)
  4. Set the gateway to your VPN interface IP (e.g., 10.0.0.1 for WireGuard)
  5. Choose aggregation mask (recommended /24) and click Analyze Website
  6. Copy the generated routes or download as .sh

Step 2. Add routes via web UI

Easiest approach — use the Keenetic web interface:

  1. Open http://my.keenetic.net or the router's IP
  2. Sign in as administrator
  3. Go to Management → Routing
  4. Click Add Route
  5. Fill in:
    • Route type: Route to remote network
    • Destination network: e.g. 104.21.32.0
    • Subnet mask: 255.255.255.0 (for /24)
    • Interface: your VPN tunnel
  6. Repeat for each subnet from NetRoute Pro
Tip: if you have 50+ routes, use CLI — much faster than manual entry.

Step 3. Bulk add via CLI

For mass import, connect to Keenetic via SSH or Telnet:

ssh admin@192.168.1.1

Enter configuration mode and add routes:

(config)> ip route 104.21.32.0/24 Wireguard0 auto
(config)> ip route 172.67.182.0/24 Wireguard0 auto
(config)> ip route 151.101.1.0/24 Wireguard0 auto
(config)> system configuration save

Replace Wireguard0 with your VPN interface name. List interfaces:

(config)> show interface

Step 4. WireGuard AllowedIPs approach

With WireGuard, you can skip static routes entirely — set AllowedIPs on the peer instead:

  1. In NetRoute Pro, select WireGuard platform
  2. Copy the AllowedIPs line
  3. In Keenetic web UI, go to Internet → Other connections → WireGuard
  4. Open peer settings and paste into the Allowed IPs field
  5. Save
AllowedIPs = 104.21.32.0/24, 172.67.182.0/24, 151.101.1.0/24

Verify

Check that routes are active:

(config)> show ip route

Verify traffic goes through VPN — from a LAN client:

tracert example.com   # Windows
traceroute example.com   # Linux/macOS

First hops should go through your VPN gateway IP.

Common issues

Route added but traffic doesn't go through VPN

  • Check the VPN interface is up: show interface Wireguard0
  • Verify there are no blocking firewall rules
  • Check NAT/masquerade on the VPN interface

Website IP changed — route stopped working

Cloudflare, Fastly, and other CDNs rotate IPs. Regenerate routes with RIPE BGP optimization enabled in NetRoute Pro — it substitutes announced BGP prefixes that cover all provider IPs.

Too many routes

Keenetic handles 500-1000 static routes without issues. For more, use WireGuard AllowedIPs or set up BGP via opkg packages.
