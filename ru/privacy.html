Политика конфиденциальности

Последнее обновление: 30 марта 2026

Обзор

NetRoute Pro — это расширение для браузера, которое обнаруживает домены на веб-страницах, резолвит их в IP-адреса и генерирует команды маршрутизации. Ваша конфиденциальность важна для нас. Это расширение работает полностью локально в вашем браузере.

Сбор данных

NetRoute Pro не собирает, не хранит, не передаёт и не делится никакими персональными данными. А именно:

  • Никаких скриптов аналитики или отслеживания
  • Никаких учётных записей или регистрации
  • Никаких cookies для отслеживания
  • Никаких данных, отправляемых на наши серверы
  • Никакой записи истории просмотров

Сетевые запросы

Расширение выполняет следующие сетевые запросы, необходимые для его основной функциональности:

  • DNS-over-HTTPS запросы к Cloudflare (1.1.1.1), Google (8.8.8.8) и Quad9 (9.9.9.9) — для резолвинга доменных имён в IP-адреса
  • RIPE Stat API (stat.ripe.net) — для получения информации о BGP-префиксах для оптимизации маршрутов (опционально)
  • ip-api.com — для поиска ASN/организации для разрешённых IP (опционально)

Эти запросы содержат только доменные имена или IP-адреса, обнаруженные на текущей веб-странице. Никакая персональная информация не включается.

Локальное хранилище

Расширение использует chrome.storage.local и опционально chrome.storage.sync для хранения:

  • Пользовательских настроек (тема, DNS-провайдер, параметры экспорта)
  • Чёрного списка доменов
  • Истории сканирований (последние 10, хранятся локально)
  • DNS и BGP кеша (с TTL истечением)

Все данные остаются на вашем устройстве (или синхронизируются через ваш аккаунт Google, если включена Chrome Sync). Мы не имеем доступа к этим данным.

Разрешения

  • activeTab — для доступа к URL текущей вкладки для сканирования доменов
  • scripting — для извлечения доменов с текущей страницы
  • storage — для сохранения настроек и кеша локально
  • downloads — для экспорта файлов маршрутов (.bat, .sh, .rsc)
  • webRequest — для перехвата сетевых запросов при обнаружении доменов
  • host_permissions (все URL) — для выполнения DNS-over-HTTPS запросов для резолвинга доменов

Сторонние сервисы

Никакие данные не передаются третьим сторонам для рекламы, маркетинга или аналитики. Единственные внешние сервисы, с которыми осуществляется связь — это DNS-провайдеры и API сетевой информации, перечисленные выше, исключительно для основной функциональности расширения.

Изменения
