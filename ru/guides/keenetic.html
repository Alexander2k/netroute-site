Добавление маршрутов на Keenetic

Пошаговая инструкция: как направить трафик отдельных сайтов через VPN-туннель на роутере Keenetic. Генерируем маршруты из доменов через NetRoute Pro и импортируем их на роутер.

Что понадобится

  • Роутер Keenetic с актуальной прошивкой KeeneticOS 3.x или выше
  • Настроенный VPN-туннель (WireGuard, OpenVPN, IKEv2 или другой)
  • Расширение NetRoute Pro в Chrome
  • Доступ к веб-интерфейсу роутера (обычно http://my.keenetic.net)

Шаг 1. Сгенерируйте маршруты через NetRoute Pro

  1. Откройте нужный сайт в браузере (например, заблокированный ресурс)
  2. Нажмите на иконку NetRoute Pro в панели расширений Chrome
  3. Выберите платформу Linux (Keenetic принимает команды в Linux-формате)
  4. Укажите шлюз — IP вашего VPN-интерфейса (например, 10.0.0.1 для WireGuard)
  5. Задайте маску агрегации (рекомендуется /24) и нажмите Analyze Website
  6. Скопируйте сгенерированные маршруты или скачайте как .sh файл

Шаг 2. Добавьте маршруты через веб-интерфейс

Самый простой способ — использовать веб-интерфейс Keenetic:

  1. Откройте http://my.keenetic.net или IP-адрес роутера
  2. Войдите под учётной записью администратора
  3. Перейдите в раздел Управление → Маршрутизация
  4. Нажмите Добавить маршрут
  5. Заполните поля:
    • Тип маршрута: Маршрут до удалённой подсети
    • Адрес сети назначения: например 104.21.32.0
    • Маска подсети: 255.255.255.0 (для /24)
    • Интерфейс: ваш VPN-туннель
  6. Повторите для каждой подсети из списка NetRoute Pro
Совет: если маршрутов много (50+), используйте CLI — это быстрее, чем добавлять вручную.

Шаг 3. Добавление через CLI (быстрый способ)

Для массового добавления подключитесь к Keenetic через SSH или Telnet:

ssh admin@192.168.1.1

Войдите в режим конфигурации и добавьте маршруты:

(config)> ip route 104.21.32.0/24 Wireguard0 auto
(config)> ip route 172.67.182.0/24 Wireguard0 auto
(config)> ip route 151.101.1.0/24 Wireguard0 auto
(config)> system configuration save

Вместо Wireguard0 подставьте имя вашего VPN-интерфейса. Посмотреть список интерфейсов:

(config)> show interface

Шаг 4. Настройка WireGuard через AllowedIPs

Если вы используете WireGuard, проще задать маршруты прямо в AllowedIPs пира — тогда отдельные ip route не нужны:

  1. В NetRoute Pro выберите платформу WireGuard
  2. Скопируйте строку AllowedIPs
  3. В веб-интерфейсе Keenetic откройте Интернет → Другие подключения → WireGuard
  4. Откройте настройки пира и вставьте список в поле Allowed IPs
  5. Сохраните настройки
AllowedIPs = 104.21.32.0/24, 172.67.182.0/24, 151.101.1.0/24

Проверка работы

Убедитесь, что маршруты применились:

(config)> show ip route

Проверьте, что сайт идёт через VPN — на компьютере в LAN:

tracert example.com   # Windows
traceroute example.com   # Linux/macOS

Первые хопы должны идти через IP вашего VPN-шлюза.

Частые проблемы

Маршрут добавлен, но трафик не идёт через VPN

  • Проверьте, что VPN-интерфейс активен: show interface Wireguard0
  • Убедитесь что в правилах файрвола нет блокировки
  • Проверьте NAT/Masquerade на VPN-интерфейсе

IP сайта изменился — маршрут не работает

Cloudflare, Fastly и другие CDN могут менять IP. Пересоздайте маршруты через NetRoute Pro с включённой RIPE BGP оптимизацией — она подставит анонсированные BGP-префиксы, которые покрывают все возможные IP провайдера.

Слишком много маршрутов

Keenetic без проблем держит 500-1000 статических маршрутов. Если нужно больше — используйте WireGuard AllowedIPs или настройте BGP через opkg-пакеты.
